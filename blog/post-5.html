FASE: Building an AI Workflow Framework with 12 Agents and 32 CLI Commands

March 17, 2026 7 min read

FASE started as a collection of scripts I kept copying between projects. It became a published npm package with 12 specialised AI agents and 32 CLI commands, runnable anywhere with npx fase-ai. Here's what I learned designing an AI-first developer tool from scratch.

The Problem: AI Tooling That Doesn't Compose

Most AI coding tools are monolithic — you get a chat interface or an autocomplete, and that's the interaction model. FASE takes a different approach: small, focused agents that each do one thing well and can be chained together into workflows. Think Unix pipes, but for AI-assisted development tasks.

The name stands for: Fetch context, Analyse, Synthesize, Execute. Every agent follows this loop — gather what it needs, reason about it, produce output, act on it.

The Agent Architecture

Each of the 12 agents is a focused specialist:

  • Planner — breaks a feature request into ordered tasks
  • Coder — generates implementation for a single task
  • Reviewer — critiques code against a defined style guide
  • Tester — writes unit and integration tests for a module
  • Documenter — generates JSDoc and README content
  • Refactorer — proposes structural improvements without changing behaviour
  • Debugger — traces a bug through call stacks and logs
  • Migrator — handles API version upgrades and dependency changes
  • Searcher — finds relevant code across the codebase
  • Summariser — produces readable summaries of large changesets
  • Validator — checks output from other agents for correctness
  • Orchestrator — coordinates multi-agent workflows

Agents communicate through a shared context object passed between steps — structured JSON that accumulates findings and artefacts as the workflow progresses.

Designing 32 CLI Commands

CLI design is UX design. A command nobody can remember doesn't get used. The naming conventions I settled on:

  • Verbs for actions: fase plan, fase review, fase test
  • Nouns for inspection: fase agents, fase workflows
  • Consistent flags: --file, --out, --model work the same across all commands

I ran the command set through a "stranger test" — could someone who'd never seen FASE infer what each command does from its name alone? Commands that failed the test were renamed or merged.

Publishing to npm

Publishing with npx fase-ai as the entry point required careful attention to the package.json bin field and making the CLI entry point executable (#!/usr/bin/env node). The zero-install experience via npx means users can try it without committing to a global install — the right default for a dev tool people want to evaluate quickly.

Versioning follows semantic versioning strictly: agent behaviour changes are breaking changes (major), new commands are features (minor), prompt refinements are patches. This discipline matters more than it might seem — developers building workflows on top of FASE need to know what to expect between updates.
