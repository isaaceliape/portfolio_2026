splitfair: Bill Splitting with Web Components and Canvas

March 15, 2026 5 min read

splitfair solves a real problem I kept running into: splitting a restaurant bill between a group without anyone pulling out a spreadsheet. The twist is the export — the app generates a shareable graphic of the split via the Canvas API so you can send it directly in a chat.

Why Web Components?

splitfair was a deliberate exercise in building without a framework. Web Components gave me encapsulated, reusable UI elements — a person input card, a bill line item, a results summary — without a build-time dependency on React or Vue. Vite handles bundling; the rest is vanilla with custom elements.

The main challenge with Web Components is styling encapsulation. Shadow DOM prevents style leakage, but it also means you can't use global CSS variables without explicitly using CSS custom properties in the component styles. I leaned into this — every component consumes a small set of design tokens exposed on :host, which made theming and consistent spacing straightforward.

Touch Optimisation

The primary use case is at a table with a phone. Every interactive element follows the 44px minimum touch target rule, inputs use numeric keyboards where appropriate, and swipe-to-remove works on person cards. The layout adapts aggressively for small screens — the step-by-step flow (add people → add items → assign → split) works on a 375px screen without horizontal scrolling.

I paid particular attention to the number inputs. On mobile, you want inputmode="decimal" rather than type="number" — it gives you the numeric keyboard without the browser spinner controls that interfere with touch targets on narrow screens.

Canvas Export

The shareable graphic is drawn programmatically using the Canvas 2D API. The process:

  1. Create an offscreen <canvas> element at 2× pixel density
  2. Draw the background, border, and result rows using fillRect and fillText
  3. Apply the same font (loaded via FontFace) as the main UI
  4. Export with canvas.toBlob() and trigger a download or Web Share API call

The Web Share API integration is the satisfying part: on mobile, tapping "Share" invokes the native share sheet directly, so the graphic goes straight to WhatsApp, iMessage, or wherever the group chat lives — no copy-paste, no screenshot.

The Split Algorithm

Equal split is trivial. The interesting case is unequal contribution — some items are shared, some are individual, tip percentage may vary. The algorithm:

  • Each item can be assigned to one or more people
  • Shared items divide evenly among assignees
  • Tip and service charges apply proportionally to each person's subtotal
  • Rounding distributes the remainder to the first person to avoid off-by-one errors

Getting rounding right is the part most bill-split apps get wrong. A €0.01 discrepancy is small, but it makes users distrust the result. Explicit remainder distribution makes it auditable.
